Reactor control readme

  • About the program

    • The program is intended to automate any ic2 reactor setup.
      The software is not yet complete; features are prioritized according to what I need for my reactor(s).

  • Requirements
    • ComputerCraft OR OpenComputers
      • Opencomputers support not complete
    • OpenPeripherals compatible inventory API
      • In particular, pushItem and pullItem, getAllStacks, getStackInSlot.
  • Hardware setup

    • REACTOR: The reactor must be set up with the component pattern you wish to use, and must be connected as a peripheral/component.

      SUPPLIER: An inventory which will contain the reactor components that will be moved into the reactor as needed. Must be connected as a peripheral/component.

      + - PULLER: An inventory which will receive components removed from the reactor. Must be connected as a peripheral/component.

      • May be the same inventory as the supplier, as long as one can prevent it getting pulled back in.. See also ME Interface.

        May receive items like

        • Spent fuel
        • Empty condensers
        • Hot cooling cells/OC vents meant for cooling elsewhere.

      + - REDSTONE SOURCE: Software controlled device emitting the redstone signal that activates the reactor.

      • With a CC computer, this will be the computer itself.
      • The reactor MUST NOT run unless  this signal is active.

      (Optional) POWER STORAGE: The software can be configured to stop the reactor if nothing will receive its power output.

      (Optional) NETWORK INTERFACE: The software will broadcast status reports on the network.

      (Optional) MONITOR: Used to display status reports
